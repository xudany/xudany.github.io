### git commit 提交类型说明

- feat : 新功能（feature）
- fix : 修复bug 
- docs : 修改了文档，比如README，CHANGELOG等等
- style : 修改了格式，例如修改了空格，格式缩进等不影响代码运行，不改变代码逻辑的变动
- refactor : 代码重构，既不是增加新功能，也不是修复bug的代码变动
- test : 测试用例，包括单元测试，集成测试等
- chore : 改变构建流程、或者增加依赖库、工具等构建过程或辅助工具的变动
